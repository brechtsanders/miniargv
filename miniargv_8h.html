Detailed Description

miniargv library header file with main functions

Author
Brecht Sanders

The miniargv library C library is a lightweight cross-platform C/C++ library for processing command line arguments and displaying command line help.

typedef int(* miniargv_complete_fn) (char *argv[], char *env[], const miniargv_definition *argdef, const miniargv_definition *envdef, const miniargv_definition *currentarg, const char *arg, int argparampos, void *callbackdata)

callback function called by miniargv_completion() to list possible parameters during bash completion

Parameters
argvNULL-terminated array of arguments (first one is undefined)
envNULL-terminated array of environment variables
argdefdefinitions of possible command line arguments
currentargdefinition of command line argument that is being completed
argargument being completed (actual parameter value starts at position argparampos)
argparamposposition within arg where the parameter starts (e.g. position of "123" in "--val=123")
callbackdatauser data as passed to miniargv_completion()
Returns
0 to continue completion, non-zero to abort (e.g. for standalone value argument return non-zero to skip short and long arguments) This function should list all possible completions on standard output, one result per line. The part of the argument preceding the value to be completed must also be included. This can be done like this (where possiblevalue is a possible value for completion):
printf("%.*s%s\n", argparampos, arg, possiblevalue);

DLL_EXPORT_MINIARGV int miniargv_process_cfgfile ( const char * cfgfile,
const miniargv_definition cfgdef[],
const char * section,
void * callbackdata )

process configuration file variables and call the appropriate callback function for each match (note: the values read are not kept in memory, so use miniargv_cb_strdup instead of miniargv_cb_set_const_str for string data and make sure free the allocated data when no longer needed)

Parameters
cfgfilepath of configuration file to read
cfgdefdefinitions of possible configuration file variables (shortarg is ignored)
sectionsection to load in additional to "default", or NULL to only load section "default"
callbackdatauser data passed to callback functions
Returns
0 on success or abort code returned by callback function A configuration file can have any extension (though .cfg or .ini is recommended). Whitespace at the beginning of a line is ignored. Any line starting with a semicolon (;) or hash sign (#) is considered as command and will be ignored. Lines starting with an at-sign (@) will cause the configuration file specified after the at-sign (@) to be processed. Configuration lines consist of a variable name, followed by an equals (=) sign or a colon (:), followed by a value. Whitespace before and after the equals (=) sign or a colon (:) is ignored. It is also possible to use an at-sign (@) instead of an equals (=) sign or a colon (:), but then the value specifies a file that will be loaded (including all white space and line breaks). Section names start with the name of the section enclosed in square brackets ([]). If no section name is specified the default section name is "default". Section names are not case sensitive.

◆ miniargv_getprogramname()

DLL_EXPORT_MINIARGV const char * miniargv_getprogramname ( const char * argv0,
int * length )

get application name and length

Gets the name of the current application from the first argv entry (argv[0]) as passed to main().

Though the result is a null-terminated string, only the number of characters pointed to by length should be used (e.g. to avoid the ".exe" extension on Windows).

NULL is returned on error or if argv0 is NULL or points to an empty string.

Parameters
argv0argv[0] as passed to main()
lengthpointer that will receive the length of the application name (can be NULL)
Returns
application name, or NULL on error
